# 2.0 Readiness Gate

Comfydex `1.9.0` adds the 2.0 Readiness Gate. It is a machine-readable report for checking whether the plugin is ready to claim the final ordinary-user 2.0 workflow system.

The gate does not claim unsupported workflows are finished. It lists the first-class scenarios, shows which ones are ready, and reports exact gaps such as `missing_recipe`, incomplete semantic coverage, or UI graph build problems.

## MCP Tools

```text
comfy_list_20_scenarios
comfy_20_readiness_report
```

Use `comfy_list_20_scenarios` to inspect the first-class scenarios required for 2.0:

- text-to-image,
- image-to-image,
- portrait,
- character-consistency,
- product-image,
- ControlNet,
- inpainting,
- upscaling,
- background-replacement.

Use `comfy_20_readiness_report` to get the full report. The report includes:

- `readiness_version`,
- overall `status`,
- scenario counts,
- per-scenario recipe coverage,
- dry-run UI graph build checks,
- 2.0 acceptance criteria,
- `next_steps`.

## Status Terms

- `ready`: at least one mapped recipe can produce a complete generation plan and valid generated UI workflow.
- `partial`: a recipe exists but still has missing information, partial semantic coverage, or UI graph build gaps.
- `missing_recipe`: the scenario is required for 2.0 but has no first-class recipe yet.
- `needs_work`: the overall gate still has scenario or release criteria gaps.
- `ready_for_2_0`: all first-class scenarios and acceptance criteria are ready.

## Desktop

The Desktop Settings view shows the same 2.0 Readiness Gate report. It displays scenario totals, ready counts, remaining gaps, each first-class scenario status, acceptance criteria, and the first few next steps.

Desktop remains a status and confirmation console. It does not become a full ComfyUI graph editor in `1.9.0`.

## Release Boundary

`1.9.0` is a readiness release. It makes remaining 2.0 gaps visible and testable. Final `2.0.0` can be claimed only after the report reaches `ready_for_2_0` and the final release validation checks pass.
