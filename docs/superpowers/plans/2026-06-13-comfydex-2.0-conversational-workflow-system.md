# Comfydex 2.0 Conversational Workflow System Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Comfydex `2.0.0` as an ordinary-user conversational workflow system whose 2.0 readiness report reaches `ready_for_2_0`.

**Architecture:** Keep the 1.9 readiness gate as the release arbiter and close the remaining gaps by adding first-class recipes and one reusable inpaint UI graph template. The generation planner, recipe registry, UI graph builder, desktop fallback data, release validator, and documentation all read the same scenario vocabulary so Codex can explain, build, and push workflows from natural-language requests without claiming silent downloads or automatic installs.

**Tech Stack:** Python MCP server, pytest, ComfyUI workflow JSON, Tauri desktop shell, TypeScript, Vite, Rust, Git.

---

## File Map

- `src/comfydex_mcp/templates.py`: add the `inpaint-basic` template and route inpaint/background intents to it.
- `src/comfydex_mcp/recipes.py`: add 2.0 scenario recipes for portrait, character consistency, product image, inpainting, and background replacement.
- `src/comfydex_mcp/generation.py`: normalize `mask` and rank inpaint/background requests correctly.
- `src/comfydex_mcp/ui_graphs.py`: add readable titles, layout columns, and fallback output types for inpaint graph nodes.
- `src/comfydex_mcp/readiness.py`: map all nine first-class scenarios to ready recipes and bump `READINESS_VERSION` to `2.0.0`.
- `tests/test_templates.py`: prove the new template exists, selects by intent, and reports missing mask information.
- `tests/test_recipes.py`: prove the recipe registry covers all 2.0 scenarios and suggests the correct recipes.
- `tests/test_generation.py`: prove natural inpaint/background requests select the inpaint template and preserve mask parameters.
- `tests/test_ui_graphs.py`: prove the inpaint plan builds a valid readable UI graph.
- `tests/test_readiness.py`: prove the readiness report reaches `ready_for_2_0`.
- `tests/test_server_tools.py`, `tests/test_desktop_bridge.py`, `tests/test_version.py`, `tests/test_docs_examples.py`: update assertions for the final 2.0 release.
- `desktop/src/lib/api.ts`, `desktop/src-tauri/src/main.rs`: update fallback readiness data and app info to `2.0.0`.
- `README.md`, `docs/usage/2.0-readiness-gate.md`, `docs/usage/desktop-app.md`, `skills/comfyui-workflows/SKILL.md`: describe the final 2.0 behavior.
- `docs/usage/conversational-workflow-system.md`: add ordinary-user 2.0 workflow system documentation.
- `docs/release/2.0-release-checklist.md`: add release checklist and Git tag commands for `v2.0.0`.
- `scripts/validate_release_package.py`: require 2.0 docs, release markers, and readiness metadata.
- Version files: `pyproject.toml`, `src/comfydex_mcp/__init__.py`, `.codex-plugin/plugin.json`, `desktop/package.json`, `desktop/package-lock.json`, `desktop/src-tauri/Cargo.toml`, `desktop/src-tauri/Cargo.lock`, `desktop/src-tauri/tauri.conf.json`.

Comfydex `2.0.0` is the first ordinary-user conversational workflow system release.

## First-Class Scenarios

The 2.0 first-class scenarios are text-to-image, image-to-image, portrait, character consistency, product image, ControlNet, inpainting, upscaling, and background replacement.

## How Codex Uses It

Codex reads the recipe registry, plans a workflow from natural language, reports missing models or images, builds a readable ComfyUI UI workflow, and can use Live Bridge to push the generated graph to the ComfyUI canvas when the bridge is available.

## Safety Boundary

Comfydex performs no automatic downloads and no automatic custom node installation. Missing models or nodes are reported through capability reports and install plans that require confirmation and audit records.

## Readiness

The 2.0 Readiness Gate must return `ready_for_2_0` before this release is claimed.
