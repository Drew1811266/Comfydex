# Comfydex 0.9 End-To-End Automation And Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Comfydex `0.9.0` with a safe one-call generate-run-fetch MCP workflow and release package validation.

**Architecture:** Add orchestration at the MCP server layer while reusing the existing generation, workflow, submit, wait, fetch, and indexing services. Harden shared submit policy reasons in `generation.py`, keep filesystem state in the existing workflow/run/output/index locations, and add a pure package validation script for release readiness.

**Tech Stack:** Python 3.11+, pytest, MCP FastMCP, existing ComfyClient abstractions, Tauri/Vite/React desktop checks, Cargo check.
