# Comfydex 0.7 Tauri Desktop Shell Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver Comfydex `0.7.0` with a Windows-first Tauri desktop app shell, a Python desktop bridge over the shared Comfydex core, React workbench views, Tauri commands, documentation, and release metadata.

**Architecture:** Add `desktop_bridge.py` as a JSON CLI over existing Python services. Add `desktop/` as an npm-managed Tauri v2 + Vite + React + TypeScript project. The frontend talks only to `desktop/src/lib/api.ts`; `api.ts` invokes Tauri commands when available and returns deterministic fallback data in browser-only builds.

**Tech Stack:** Python 3.11, pytest, Node/npm, Vite, React, TypeScript, Tauri v2, Rust.
