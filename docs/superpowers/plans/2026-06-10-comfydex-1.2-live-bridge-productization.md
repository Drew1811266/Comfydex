# Comfydex 1.2 Live Bridge Productization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Comfydex `1.2.0` with a dependable, user-visible Live Bridge path: install or upgrade the ComfyUI custom node, verify ComfyUI and bridge readiness, push a UI workflow into the running canvas, surface actionable errors, and show bridge status in Comfydex Desktop.

**Architecture:** Keep the ComfyUI custom node as the runtime transport, add frontend heartbeat and workflow acknowledgement routes to make canvas delivery observable, add a Python `live_bridge.py` service layer over `ComfyClient`, expose the service through FastMCP tools and the desktop bridge, then wire the Tauri/React desktop app to that bridge. Installer and verification scripts remain Windows-first PowerShell entry points that share the same route contract.

**Tech Stack:** Python 3.12, FastMCP, httpx, pytest, respx, ComfyUI `PromptServer` aiohttp routes, ComfyUI frontend extension JavaScript, PowerShell 7, Tauri v2, React 19, TypeScript, Vite, Rust 2021.

---

## Scope Guard

1.2.0 productizes the existing Live Bridge. It does not implement the 1.3 Node Semantic Registry, natural language recipe planning, model installation planning, or workflow graph generation beyond pushing already-built UI workflow JSON.

The 1.2.0 exit line is:

- ComfyUI can be checked from Codex and Desktop.
- Live Bridge install or upgrade is safe and explainable.
- Bridge backend, frontend loader, frontend client, and route versions are visible.
- Push requests can report whether the frontend accepted, refused, or did not acknowledge the workflow.
- A normal user can install, restart once, verify, and receive a pushed workflow without reading source code.

## Shared Status Contract

All Python tools, scripts, and Desktop views use this normalized shape:

```json
{
  "ok": true,
  "ready": true,
  "base_url": "http://127.0.0.1:8188",
  "checked_at": "2026-06-10T00:00:00+00:00",
  "server": {
    "reachable": true,
    "status_code": 200
  },
  "bridge": {
    "loaded": true,
    "name": "comfydex_live_bridge",
    "version": "1.2.0",
    "generation": 2,
    "routes": [
      "status",
      "load_workflow",
      "reload_client",
      "reload_backend",
      "frontend_status",
      "workflow_result"
    ]
  },
  "frontend": {
    "listed": true,
    "connected": true,
    "stale": false,
    "version": "bootstrap",
    "client_id": "client-1",
    "last_seen_at": "2026-06-10T00:00:00+00:00",
    "last_seen_age_ms": 500
  },
  "can_push": true,
  "needs_restart": false,
  "needs_refresh": false,
  "diagnostics": []
}
```

Failure diagnostics must use stable codes:

- `comfyui_unreachable`
- `bridge_not_loaded`
- `bridge_status_not_ok`
- `bridge_frontend_not_listed`
- `frontend_not_connected`
- `frontend_stale`
- `unsaved_canvas`
- `stale_client`
- `workflow_ack_timeout`
- `workflow_must_be_json_object`
- `workflow_not_ui_json`
