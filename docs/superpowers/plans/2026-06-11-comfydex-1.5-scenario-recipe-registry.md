# Comfydex 1.5 Scenario Recipe Registry Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Comfydex `1.5.0` with a scenario recipe registry that lets Codex map natural workflow intent to explainable recipes, workflow templates, required inputs, model requirements, node requirements, and capability checks.

**Architecture:** Add a focused recipe layer above existing workflow templates and below natural-language planning. Recipes describe user-facing scenarios such as text-to-image, image-to-image, LoRA text-to-image, ControlNet pose, and upscale, then map those scenarios to existing templates and capability requirements. The recipe layer must enrich planning and capability reports without generating UI graphs, downloading assets, or installing custom nodes.

**Tech Stack:** Python 3.11, pytest, FastMCP, existing `templates.py`, existing `generation.py`, existing `capabilities.py`, React/Tauri desktop bridge wrappers only where needed, Markdown docs.

---

## Scope Boundary

1.5 implements:

- a built-in scenario recipe registry,
- list/search/explain/suggest recipe APIs,
- natural-language recipe scoring,
- recipe-to-template mapping,
- recipe-aware capability resolution,
- MCP and desktop bridge operations for recipe discovery and capability checks,
- recipe context in workflow generation plans,
- docs, tests, and version metadata for `1.5.0`.

1.5 does not implement:

- UI workflow graph generation,
- automatic model downloads,
- automatic custom node installation,
- large third-party recipe marketplaces,
- user-authored recipe persistence,
- multi-agent prompt decomposition.

```python
def suggest_workflow_recipes(
    intent: str,
    parameters: dict[str, Any] | None = None,
    *,
    recipe_id: str | None = None,
    limit: int = 3,
) -> list[dict[str, Any]]
```

Scoring rules:

- explicit `recipe_id` returns score `1000`,
- intent phrase exact substring match adds `80`,
- tag substring match adds `35`,
- required model parameter presence adds `45` for matching model types,
- required input parameter presence adds `20`,
- recipe name word match adds `10` per matched word.

Return sorted descending by score, then ascending recipe id.
