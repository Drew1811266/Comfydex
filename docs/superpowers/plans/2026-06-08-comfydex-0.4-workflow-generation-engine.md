# Comfydex 0.4 Workflow Generation Engine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver Comfydex `0.4.0` with deterministic workflow generation planning, validation/repair results, submit policy classification, MCP tools, documentation, and release metadata.

**Architecture:** Add `generation.py` as the engine layer above templates, builder, validation, workflow persistence, and the `0.3.0` project index. Existing builder tools stay compatible and can delegate to the generation engine. MCP tools expose plan, generate, and policy operations without adding an external LLM dependency.

**Tech Stack:** Python 3.11, existing FastMCP server, existing pytest suite, deterministic template catalog, existing ComfyUI object-info validation.
