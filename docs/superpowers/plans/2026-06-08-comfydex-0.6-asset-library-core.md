# Comfydex 0.6 Asset Library Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver Comfydex `0.6.0` with a shared asset library core: schema version `2`, searchable asset records, user annotations, sidecar metadata, asset cleanup planning, asset reports, asset comparison, MCP tools, documentation, and release metadata.

**Architecture:** Extend the shared project core instead of adding a separate store. `core/schema.py`, `core/store.py`, and `core/indexer.py` own migration and deterministic asset indexing. New `assets.py` owns search, annotation, sidecars, cleanup plans, reports, and asset comparisons. `server.py` exposes asset MCP tools through the same `ProjectContext` used by prior project-index tools.

**Tech Stack:** Python 3.11, SQLite, existing shared core, existing FastMCP server, pytest, Windows-first path safety helpers.
