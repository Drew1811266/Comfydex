# Comfydex 1.4 Capability Resolver And Install Planner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Comfydex `1.4.0` with environment-aware capability reports and conservative install plans before workflow generation relies on local nodes or model assets.

**Architecture:** Add a Python capability layer that scans model roots, reads live `object_info`, maps workflow plans to required nodes/models, and produces a run-ready/missing/blocked report. Add a conservative install plan format and append-only audit log, expose the capability layer through MCP and desktop bridge operations, then surface pending install plan review in the desktop Settings view. Keep scenario recipe selection, automated downloads, and UI graph generation out of 1.4.

**Tech Stack:** Python 3.11, pytest, FastMCP, existing `ComfyClient.get_object_info`, existing `plan_workflow_generation`, React/TypeScript/Tauri desktop shell, Markdown docs.

---

## Scope Boundary

1.4 implements:

- local model inventory with filename and model-type inference,
- live node inventory from ComfyUI `object_info`,
- capability report for a requested workflow plan,
- missing model and missing node reports,
- conservative install plan data format,
- append-only install audit log,
- MCP tools and desktop bridge operations for capability/install plan review,
- a desktop confirmation surface for reviewing install plans.

1.4 does not implement:

- silent model downloads,
- automatic custom node installation,
- scenario recipe library,
- generated UI workflow graphs,
- one-click retry repair loops.
