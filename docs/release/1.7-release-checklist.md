# Comfydex 1.7 Release Checklist

Comfydex 1.7 ships the Execution And Repair Loop for failed generated and submitted runs, MCP repair tools, automation failure repair payloads, desktop bridge repair operations, and the Runs repair panel.

## Automated Gates

- `python -m pytest -q`
- repair planner tests: `tests/test_repairs.py`
- MCP repair tools: `tests/test_server_tools.py`
- automation failure repair payloads: `tests/test_server_tools.py -k generate_run_fetch`
- desktop bridge repair operations: `tests/test_desktop_bridge.py`
- Runs repair panel: `desktop/src/views/RunsView.tsx`
- docs/version checks: `tests/test_docs_examples.py tests/test_version.py`
- `python scripts\validate_release_package.py`
- `npm run typecheck`
- `npm run build`
- `cargo check`

## Release Criteria

- `comfy_diagnose_run` exposes `failure_class`, `repair_summary`, and `retryable`.
- `comfy_plan_run_repair` returns diagnosis and a structured `repair_plan`.
- `comfy_retry_run_repair` can retry `fetch_outputs` without confirmation and blocks confirmation-required resubmits until confirmed.
- `comfy_read_repair_history` returns newest repair history entries first.
- `comfy_generate_run_fetch` failed submit, wait, failed wait, and fetch responses include repair payloads.
- The Python desktop bridge supports `plan_run_repair`, `retry_run_repair`, and `read_repair_history`.
- The desktop Runs repair panel displays failure class, repair summary, actions, retry state, and recent history.
- The release keeps no silent downloads, no automatic downloads, no automatic custom node installation, and no unconfirmed resubmission.
- version files report `1.7.0`.

## Integration

After all gates pass, merge the feature branch into `main`, push `main`, create annotated tag `v1.7.0`, push the tag, and verify:

```powershell
git ls-remote origin refs/heads/main refs/tags/v1.7.0 "refs/tags/v1.7.0^{}"
```
