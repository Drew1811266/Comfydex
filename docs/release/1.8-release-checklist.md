# Comfydex 1.8 Release Checklist

Comfydex 1.8 ships Ordinary User Polish for common text-to-image work: plain user guidance helpers, generation presets, safer canvas replacement summaries, output asset summaries, desktop bridge support, and desktop ordinary-user UI polish.

## Automated Gates

- `python -m pytest -q`
- plain user guidance helpers: `tests/test_user_guidance.py`
- generation presets: `tests/test_presets.py tests/test_generation.py`
- MCP user guidance tools: `tests/test_server_tools.py`
- asset summaries and comparison language: `tests/test_assets.py`
- Live Bridge canvas replacement guidance: `tests/test_live_bridge_user_guidance.py`
- desktop bridge user guidance operations: `tests/test_desktop_bridge.py`
- desktop ordinary-user UI polish: `desktop/src/App.tsx desktop/src/views`
- Browser desktop and mobile verification: Dashboard, Generated, and Assets views at desktop width and 390px mobile width
- docs/version checks: `tests/test_docs_examples.py tests/test_version.py`
- `python scripts\validate_release_package.py`
- `npm run typecheck`
- `npm run build`
- `cargo check`

## Release Criteria

- `comfy_plan_workflow_generation` includes `user_guidance` and `resolved_defaults`.
- `comfy_list_generation_presets` returns quality, speed, aspect ratio, and style preset groups.
- `comfy_explain_user_plan` can produce the same plain-language plan summary from an existing plan.
- `comfy_generate_run_fetch` includes `user_guidance` and successful runs include `output_summary`.
- `comfy_generate_push_ui_workflow` includes `canvas_replacement`.
- `comfy_summarize_assets` returns asset rows plus a plain summary.
- `comfy_compare_assets` includes a plain comparison summary.
- The Python desktop bridge supports `list_generation_presets` and `summarize_assets`.
- The desktop Project, Generated, Runs, and Assets views surface ordinary-user summaries without hiding technical details.
- The release keeps no silent downloads, no automatic downloads, no automatic custom node installation, and no unconfirmed resubmission.
- version files report `1.8.0`.

## Integration

After all gates pass, merge the feature branch into `main`, push `main`, create annotated tag `v1.8.0`, push the tag, and verify:

```powershell
git ls-remote origin refs/heads/main refs/tags/v1.8.0 "refs/tags/v1.8.0^{}"
```
