# Comfydex 1.0 Release Checklist

Use this checklist before pushing `v1.0.0`.

## Release Gate

Run from the repository root:

```powershell
python -m pytest tests -q
python scripts\validate_plugin.py
python scripts\validate_release_package.py
python -m json.tool .codex-plugin\plugin.json > $null
python -m json.tool .mcp.json > $null
npm --prefix desktop run typecheck
npm --prefix desktop run build
cargo check --manifest-path desktop\src-tauri\Cargo.toml
git diff --check
```

All commands must exit 0 before merge and again after merge to `main`.

## Review

Confirm:

- README current version is `1.0.0`.
- `.codex-plugin/plugin.json`, `pyproject.toml`, desktop npm metadata, Tauri metadata, and Cargo metadata are all `1.0.0`.
- `scripts/install_windows.ps1` exists and is non-destructive.
- `docs/release/windows-install.md` matches the actual install commands.
- `docs/release/security-path-review.md` covers path traversal, header redaction, cleanup confirmation, and desktop bridge boundaries.
- `python scripts\validate_release_package.py` passes.

## Merge And Tag

After feature branch verification:

```powershell
git switch main
git merge --no-ff feature/comfydex-1.0 -m "chore: merge Comfydex 1.0.0"
python -m pytest tests -q
python scripts\validate_release_package.py
git push origin main
git tag v1.0.0
git push origin v1.0.0
git ls-remote origin refs/heads/main refs/tags/v1.0.0
```

The `git ls-remote origin refs/heads/main refs/tags/v1.0.0` output must show the same commit SHA for `main` and `v1.0.0`.

## Known 1.0 Non-Goals

Comfydex `1.0.0` does not include a signed installer, bundled Python runtime, offline dependency bundle, auto-updater, ComfyUI frontend extension, model marketplace, or cloud fleet management.
