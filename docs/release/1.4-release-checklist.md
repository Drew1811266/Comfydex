# Comfydex 1.4 Release Checklist

Use this checklist before tagging `v1.4.0`.

## Repository

- Start from a clean worktree or document every intentional local change.
- Confirm the release branch contains only 1.4 Capability Resolver and Install Planner work.
- Confirm version files report `1.4.0` for Python, Codex plugin manifest, desktop package, package lock, Tauri `Cargo.toml`, Tauri `tauri.conf.json`, and `Cargo.lock`.

## Automated Checks

Run Python tests:

```powershell
python -m pytest -q
```

Run capability resolver coverage:

```powershell
python -m pytest tests\test_capabilities.py tests\test_server_tools.py tests\test_desktop_bridge.py -q
```

Run desktop checks:

```powershell
npm run typecheck
npm run build
cargo check
```

Run release package validation:

```powershell
python scripts\validate_release_package.py
```

## Manual Checks

- Confirm `comfy_model_inventory` reports model inventory files and missing roots without failing.
- Confirm `comfy_resolve_capabilities` uses live `object_info` to report node inventory, missing models, and missing nodes.
- Confirm `comfy_create_install_plan` returns a conservative install plan with `automatic: false`.
- Confirm `comfy_record_install_audit` and `comfy_read_install_audit` update the workspace audit log.
- Confirm there are no silent downloads and no automatic custom node installation.
- Confirm the desktop Install Plan panel shows status, missing requirements, actions, and audit log entries.

## Release

- Merge the release branch into `main` only after the automated gates pass.
- Push `main` to GitHub.
- Tag the release as `v1.4.0`.
- Push tag `v1.4.0` to GitHub.
