# Comfydex 1.5 Release Checklist

Comfydex 1.5 ships the Scenario Recipe Registry for recipe-aware planning and recipe-aware capability checks.

## Automated Gates

- `python -m pytest -q`
- recipe registry tests: `tests/test_recipes.py`
- MCP recipe tools: `tests/test_server_tools.py`
- desktop bridge recipe operations: `tests/test_desktop_bridge.py`
- recipe-aware planning: `tests/test_generation.py`
- recipe-aware capability checks: `comfy_resolve_recipe_capabilities`
- docs/version checks: `tests/test_docs_examples.py tests/test_version.py`
- `python scripts\validate_release_package.py`
- `npm run typecheck`
- `npm run build`
- `cargo check`

## Release Criteria

- `comfy_suggest_workflow_recipes` returns explainable recipe candidates.
- `comfy_resolve_recipe_capabilities` combines recipe requirements with live `object_info` and local model inventory.
- Generated workflow plans expose `recipe_candidates` and selected recipe id.
- Explicit `template_id` remains authoritative.
- The release keeps no automatic downloads and no automatic custom node installation.
- version files report `1.5.0`.

## Integration

After all gates pass, merge the feature branch into `main`, push `main`, create annotated tag `v1.5.0`, push the tag, and verify:

```powershell
git ls-remote origin refs/heads/main refs/tags/v1.5.0 "refs/tags/v1.5.0^{}"
```
