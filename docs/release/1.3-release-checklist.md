# Comfydex 1.3 Release Checklist

Use this checklist before tagging `v1.3.0`.

## Repository

- Start from a clean worktree or document every intentional local change.
- Confirm the release branch contains only 1.3 Node Semantic Registry Foundations work.
- Confirm version files report `1.3.0` for Python, Codex plugin manifest, desktop package, package lock, Tauri `Cargo.toml`, Tauri `tauri.conf.json`, and `Cargo.lock`.

## Automated Checks

Run Python tests:

```powershell
python -m pytest -q
```

Run registry validation:

```powershell
python -m pytest tests\test_node_semantics.py -q
```

Run object_info matching coverage:

```powershell
python -m pytest tests\test_node_semantics.py::test_match_semantics_to_object_info_reports_supported_missing_and_unknown -q
```

Run MCP registry tools coverage:

```powershell
python -m pytest tests\test_server_tools.py::test_comfy_list_node_semantics_tool_returns_supported_nodes tests\test_server_tools.py::test_comfy_explain_node_semantics_tool_refuses_unknown_node tests\test_server_tools.py::test_comfy_search_node_semantics_tool_is_registered_with_mcp tests\test_server_tools.py::test_comfy_validate_node_semantics_tool_uses_live_object_info -q
```

Run unknown-node refusal coverage:

```powershell
python -m pytest tests\test_node_semantics.py::test_get_node_semantics_returns_none_for_unknown_nodes tests\test_server_tools.py::test_comfy_explain_node_semantics_tool_refuses_unknown_node -q
```

Run desktop checks:

```powershell
npm run typecheck
npm run build
cargo check --manifest-path src-tauri\Cargo.toml
```

## Manual Checks

- Confirm `comfy_list_node_semantics` returns native nodes including `CheckpointLoaderSimple` and `KSampler`.
- Confirm `comfy_explain_node_semantics` returns `status: unsupported` for an unknown node.
- Confirm `comfy_search_node_semantics` finds sampling, LoRA, ControlNet, upscale, and inpaint-related entries.
- Confirm `comfy_validate_node_semantics` reports live unknown node types instead of claiming first-class support.

## Release

- Merge the release branch into `main` only after the automated gates pass.
- Push `main` to GitHub.
- Tag the release as `v1.3.0`.
- Push tag `v1.3.0` to GitHub.
